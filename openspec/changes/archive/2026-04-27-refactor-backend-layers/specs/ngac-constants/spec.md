## ADDED Requirements

### Requirement: NGAC operations defined as typed constants
The system SHALL define all NGAC access operations as Go constants in a single shared package (`ngac-platform/ngac`). Services MUST NOT use raw string literals for operations.

#### Scenario: Operation used in access check
- **WHEN** a service calls `policyRead.CheckAccess` with an operation
- **THEN** the operation value MUST come from the `ngac` package constants (e.g., `ngac.OpRead`, `ngac.OpWrite`)

#### Scenario: New operation added
- **WHEN** a developer needs to add a new operation (e.g., `"delete"`)
- **THEN** they SHALL add it as a constant in `ngac/ngac_ops.go` and all services pick it up via import

### Requirement: Well-known node names defined as constants
The system SHALL define well-known NGAC node names (`PC_Global`, `PublicUsers`) as constants. Services MUST NOT hardcode these names as string literals.

#### Scenario: Global PC lookup
- **WHEN** a service needs to find `PC_Global`
- **THEN** it MUST use `ngac.NodePCGlobal` constant instead of the string `"PC_Global"`

### Requirement: Workspace naming conventions as functions
The system SHALL provide functions for workspace NGAC naming patterns (e.g., `ChannelsOAName(wsName)`, `MembersUAName(wsName)`). Services MUST NOT use `fmt.Sprintf` with hardcoded patterns for workspace node names.

#### Scenario: Creating workspace Channels OA
- **WHEN** a workspace is created and needs a Channels OA node
- **THEN** the node name MUST be generated by `ngac.ChannelsOAName(wsName)` rather than `fmt.Sprintf("%s_Channels", wsName)`

#### Scenario: Finding workspace Members UA by name
- **WHEN** a service needs to find the Members UA of a workspace
- **THEN** it MUST compare against `ngac.MembersUAName(wsName)` rather than hardcoded format string

### Requirement: Access decision values defined as constants
The system SHALL define access decision values (`ALLOW`, `DENY`) as constants. Services MUST NOT compare against raw strings.

#### Scenario: Checking access result
- **WHEN** a service receives a `CheckAccess` response
- **THEN** it MUST compare `resp.Decision` against `ngac.DecisionAllow` or `ngac.DecisionDeny`
