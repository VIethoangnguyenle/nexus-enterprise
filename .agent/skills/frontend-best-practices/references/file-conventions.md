# File Conventions — TanStack Router (File-Based Routing)

## Route Directory Structure

All routes live under `frontend/src/routes/`.

```
src/routes/
├── __root.tsx                    # Root layout (required)
├── _auth.tsx                     # Auth layout route (pathless)
├── _auth/
│   ├── login.tsx                 # /login
│   └── register.tsx              # /register
├── _workspace.tsx                # Workspace layout route (pathless)
└── _workspace/
    ├── documents.tsx             # /documents
    ├── assets.tsx                # /assets
    ├── assets_.$assetId.tsx      # /assets/:assetId (dynamic param)
    ├── asset-dashboard.tsx       # /asset-dashboard
    ├── asset-types.tsx           # /asset-types
    ├── asset-requests.tsx        # /asset-requests
    ├── asset-request/
    │   └── new.tsx               # /asset-request/new
    ├── channels.$channelId.tsx   # /channels/:channelId
    └── settings.tsx              # /settings
```

## Naming Rules

### Layout Routes (Pathless)
- Prefix with `_` → `_auth.tsx`, `_workspace.tsx`
- These render `<Outlet />` and do NOT add a URL segment
- Use for shared layout, auth guards, data loading

### Dynamic Params
- Use `$paramName` → `assets_.$assetId.tsx` renders at `/assets/:assetId`
- Access via `Route.useParams()` — NOT `useParams()` from router

### Nested Directories
- Create a directory for deep nesting → `asset-request/new.tsx` → `/asset-request/new`
- Adjust import paths accordingly (3 levels deep = `../../../hooks/`)

### Route Separator
- Use `.` to separate segments → `channels.$channelId.tsx` → `/channels/:channelId`
- Use `_` to create pathless layouts

## Special Files

| File | Purpose |
|------|---------|
| `__root.tsx` | Root layout, global providers |
| `routeTree.gen.ts` | Auto-generated by TanStack Router plugin (DO NOT EDIT) |

## Rules

1. **Every route file MUST export `Route`** via `createFileRoute` or `createRootRoute`
2. **Never manually edit `routeTree.gen.ts`** — it's auto-generated by `@tanstack/router-plugin`
3. **Route path string MUST match file path** — if the file is `_workspace/assets.tsx`, the route path is `/_workspace/assets`
4. **Use directory structure for deeply nested routes** — not excessively long filenames
